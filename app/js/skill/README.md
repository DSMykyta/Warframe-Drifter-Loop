# Skill — Інструментарій для написання контенту

Ця папка містить все щоб Claude AI (або людина) могла писати повноцінні квести, діалоги і сцени для гри.

## Файли

| Файл | Що в ньому |
|------|-----------|
| **ENGINE_API.md** | Повний довідник API рушія — всі функції, типи нод, формати даних |
| **RULES.md** | Правила і обмеження — що можна, що не можна, конвенції імен |
| **LOCATIONS.md** | Таблиця всіх локацій з ID, назвами, хто де живе |
| **QUEST_TEMPLATE.js** | Шаблон квесту — копіюй і заповнюй |
| **DIALOGUE_TEMPLATE.js** | Шаблон діалогу — 4 варіанти (titles, forced, timed, з меню) |
| **EXAMPLES.md** | 5 прикладів різної складності — від простої розмови до романтичної гілки |

## Як створити новий контент

1. Визнач тип: сцена (`scenes/`) або діалог (`dialogues/`)
2. Скопіюй відповідний шаблон
3. Заповни своїм контентом
4. Збережи в правильну папку
5. `cd app/js && bash generate_manifest.sh`
6. Перезавантаж гру

## Для Claude AI

Коли користувач описує ідею квесту або діалогу:

1. Прочитай **ENGINE_API.md** для доступних функцій
2. Прочитай **RULES.md** для обмежень
3. Прочитай **LOCATIONS.md** для ID локацій
4. Використай **QUEST_TEMPLATE.js** або **DIALOGUE_TEMPLATE.js** як основу
5. Подивись **EXAMPLES.md** для стилю написання
6. Створи .js файл в правильній папці
7. Оновити content.json через generate_manifest.sh

## Стиль діалогів

- Репліки короткі. 1-2 речення максимум.
- Без емодзі. Без "(сміється)". Без зірочок *дій*.
- Персонаж говорить як живий — не пояснює очевидне.
- Гравець (mc) — мало слів. Дії через вибори.
- Наратор (null) — лаконічно. Факти, не опис.
- Телепатія (el) — уривчасто. Без повних речень.
