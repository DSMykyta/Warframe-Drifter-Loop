## 1. ЗАГАЛЬНА ОЦІНКА

Архітектура **сильна для Ren'Py проекту**. Condition-based dialogue system — нестандартне і елегантне рішення, яке уникає класичної проблеми "спагетті-діалогів". Системи між собою зв'язані логічно. Є конкретні баги та архітектурні проблеми, перелічені нижче.

```python

# gifts.rpy
def add_gossip(source, gossip_type, detail=""):
```

Друга визначення перезаписує першу (Python init order). **Фікс:** Об'єднати в одну функцію або перейменувати одну з них. Уніфікувати структуру gossip-об'єктів.

### 2.2 BUG: `check_flirt_consequences()` завжди тригериться

```python
if get_chars_at(store.current_location):

```

## 3. СЕРЙОЗНІ ПРОБЛЕМИ (зіпсують баланс/UX)

### 3.1 Mission neglect штрафує двічі

**Файл:** `dispatcher.rpy:483-494`

```python
if d >= 5:
    for name in store.chemistry:
        store.chemistry[name] = max(0, store.chemistry[name] - 5)
elif d == 4:
    for name in store.chemistry:
        store.chemistry[name] = max(0, store.chemistry[name] - 5)
```

Штраф за `d >= 5` і `d == 4` **однаковий**: -5 всім. Але `d >= 5` працює **кожен день** починаючи з п'ятого. За 10 днів без місій гравець втратить 5×6 = 30 хімії з **кожним** NPC (30 хімії × 6 персонажів = 180 пунктів). Це **знищує** будь-який прогрес і робить win condition неможливим.

Разом з `apply_decay()` (-1/-2/-3 на день per NPC), сумарний штраф стає нездоланним.

**Фікс:** Зробити `d >= 5` одноразовим через флаг, або зменшити штраф.

---

### 3.2 Sleep penalty занадто м'який — не впливає

**Файл:** `day_logic.rpy:12-13`

```python
penalty = max(0, store.minutes - 1440)  # хвилини після 24:00
wake_up = min(720, 480 + penalty)       # кеп на 12:00
```

Якщо гравець лягає рівно о 24:00 (1440 хв), penalty = 0, wake_up = 480 (08:00). Але документація обіцяє:
- 01:00 → 10:00 (2-year penalty)
- 02:00+ → 11:00

Фактично: penalty = minutes - 1440. Лягаєш о 01:00 (1500 хв) → penalty = 60 → wake_up = 540 (09:00). Це лише година пенальті, а не дві.

**Фікс:** Якщо хочете сильніший штраф, множити: `penalty = max(0, (store.minutes - 1440)) * 2`.

---

### 3.3 Decay застосовується до ВСІХ NPC одночасно, навіть свіжих

**Файл:** `dispatcher.rpy:395-407`

`apply_decay()` інкрементує `days_since_interaction` **для всіх** NPC кожен день (рядок 407). Це означає: якщо гравець не поговорив з усіма 6 NPC кожного дня — decay почне працювати. При 6 NPC і обмеженому часі (08:00-24:00 = 16 годин = 960 хвилин) — поговорити з усіма + місія + подорожі фізично неможливо кожен день.

Win condition вимагає 120+ хімії з **усіма 6**. Decay починається з дня 5, але 30-денний цикл дуже стислий.

**Фікс:** Grace period після кожної взаємодії (2-3 дні замість 0), або decay тільки для NPC з chemistry > 30.

### 3.6 Banter не ставить флаги `_saw_bruises` / `_saw_injury`

Banter entries мають `flag_false: ["amir_saw_bruises"]` etc., але `get_banter()` тільки додає ID в `seen_dialogues`. Самі флаги (`amir_saw_bruises`, `aoi_saw_injury` тощо) **ніде не ставляться**. Вони скидаються щоденно в `day_logic.rpy:45-47`, але ніколи не встановлюються.

## 4. АРХІТЕКТУРНІ ЗАУВАЖЕННЯ

### 4.1 Кирилічні ключі в даних — ризик

Імена персонажів як ключі (`"Артур"`, `"Квінсі"`) — зручно для UI, але **небезпечно** для:
- `set_flag("dating_" + name.lower())` — кирилиця в lower() може поводитись непередбачувано
- `name.lower().replace("і", "i")` в `loop_restart.rpy:170` — **хакерський фікс**, який зламається на інших літерах
- Серіалізація в save files

**Рекомендація:** Ввести `CHAR_ID` маппінг (`"Артур" → "arthur"`) і використовувати ASCII id для флагів/ключів, а кирилицю тільки для відображення.

---

### 4.2 `DIALOGUE_ENTRIES` — глобальний список без валідації

Діалоги додаються через `DIALOGUE_ENTRIES.append()` в різних файлах. Немає:
- Перевірки на дублікати `id`
- Перевірки що `label` існує
- Перевірки що `who` є в `CAST`
- Типізації полів

При зростанні контенту це стане основним джерелом багів.

**Рекомендація:** Додати `validate_entries()` функцію, яку викликати при `init`:
```python
def validate_entries():
    ids = set()
    for e in DIALOGUE_ENTRIES:
        assert e["id"] not in ids, "Duplicate: " + e["id"]
        ids.add(e["id"])
        assert renpy.has_label(e["label"]), "Missing label: " + e["label"]
```

### 4.3 Час рахується по 5 хвилин всюди — зайвий шум

Кожна репліка діалогу займає `advance_time(5)`. При 10 репліках = 50 хвилин розмови. 6 персонажів × 50 хв = 300 хвилин = 5 годин тільки на розмови. Плюс подорожі (5-10 хв кожна). Гравець фізично не встигне поговорити з усіма за день.

**Рекомендація:** Зменшити час діалогу до 2-3 хв за репліку, або рахувати час **за діалог** а не за репліку.

`can_rank_up()` існує, але **ніде не викликається**. `syndicate_rank` ніколи не підвищується автоматично. Пороги визначені (100, 300, 600, 1000, 1500 rep), але немає UI чи тригера для підвищення рангу. Rank-gated діалоги (`rank_min: 2+`) ніколи не стануть доступними.

**Фікс:** Додати перевірку `can_rank_up()` в `next_day()` або в `on_mission_complete()`, і автоматично підвищувати ранг. Або додати UI кнопку.

## 5. WIN CONDITION АНАЛІЗ (МАТЕМАТИКА)

### Чи можливо виграти за 30 днів?

**Потрібно:** 120 хімії × 6 NPC = 720 пунктів сумарно.

**Джерела хімії:**
| Джерело | За раз | Ліміт | Max за 30 днів |
|---------|--------|-------|---------------|
| Місія з напарником | +10 | 1/день/NPC | 300 (якщо чергувати) |
| Улюблений подарунок | +20 | 1/день/NPC | 600 (якщо гроші є) |
| Діалог (вибір) | +1..+3 | ~1/день/NPC | ~180 |
| Help request | +2 | 1/день total | 60 |
| Обіцянка | +5 | залежить | ~50 |

**Втрати:**
| Джерело | За раз | Частота |
|---------|--------|---------|
| Decay (5+ днів) | -1..-3/день/NPC | щоденно після 5 днів |
| Mission neglect | -3..-5 × 6 NPC | щоденно після 3 днів |
| Broken promise | -5 | per event |

**Висновок:** Теоретично можливо, але **дуже тісно**. Основна проблема — decay починається з дня 5 для **кожного** NPC окремо, і при 6 персонажах неможливо підтримувати взаємодію з усіма кожні 4 дні + робити місії + заробляти на подарунки.

**Рекомендація:** Зменшити decay threshold до 7 днів, або робити decay тільки після chemistry > 20 (не наказувати гравцю спілкуватися з тими з ким ще не подружився).
